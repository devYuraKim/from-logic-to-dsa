# DP의 Core Concepts와 Time/Space Trade-off

DP(동적계획법)의 본질은 **중복되는 부분 문제를 한 번만 계산하고, 그 결과를 재사용**하는 것.    
이 과정에서 **시간과 공간** 사이의 트레이드오프가 발생.   
**피보나치 수열**이 이 구조를 잘 보여주는 예시.


## 1. Memoization의 Time/Space Trade-off

"시간/공간 복잡도 중 무엇을 우선하겠습니까?"라는 질문.

**[DP의 핵심]**  

**한 번 계산한 값을 저장**. 단, **얼마나** 저장할 것인가에 따라 **시/공간 복잡도**가 상이.

| 전략            | 시간 복잡도             | 공간 복잡도 |
|----------------|----------------------|--------|
| 저장 안 함       | O(n) per call        | O(1)   |
| 전체 저장 (배열)  | O(1) after first call | O(n)  |


> **판단 기준**     
> - "다음번 호출 때 $O(1)$로 바로 꺼내 쓰겠다" → 전체 저장
> - "메모리를 아끼겠다, 매번 루프 한 번 도는 건 감수하겠다" → 변수 2개 유지

## 2. 저장X — 재귀 vs 반복

"저장 안 하면 매번 처음부터 계산"

- **재귀(Recursion) + 저장 없음**: 
  - 동일한 값을 중복 계산하는 호출이 기하급수적으로 증가.
  - 시간 복잡도: **$O(2^n)$** → 💀


- **반복(Iteration) + 저장 없음**:   
    - $0$부터 $n$까지 루프를 한 번 돌면 끝.
    - 시간 복잡도: 호출할 때마다 **$O(n)$**

Recursion에서 Memoization이 없으면 **지수 시간**으로 `StackOverflow` 발생.   
(그런데 아무리 Memoization 해도 중복 계산을 막아 시간 복잡도를 $O(n)$으로 낮춰줄 뿐, 재귀가 깊어지면 `StackOverflow` 발생)

## 3. 저장O — Memoization vs Tabulation

"중복 계산을 피하기 위해 결과를 기록하고 재사용"

- **재귀 + Memoization (Top-Down)**:
  - **방향**: 위에서 아래로 (큰 문제 → 작은 문제 호출).  
  - **저장**: 재귀 호출 과정에서 **필요한 값만** 계산하여 배열에 채움 (Lazy Evaluation).  
  - **장점**: 모든 상태를 방문할 필요가 없는 문제에서 효율적.
  - **단점**: 재귀가 깊어지면 `StackOverflow` 위험.


- **반복 + Tabulation (Bottom-Up)**:     
  - **방향**: 아래에서 위로 (작은 문제 → 큰 문제 순차 진행).
  - **저장**: 루프를 돌며 배열을 **처음부터 빈칸 없이** 채움 (Eager Evaluation).
  - **장점**: 모든 값을 순차적으로 채우므로 스택 오버플로우 위험이 없고 성능이 안정적.
  - **단점**: 결과에 필요 없는 하위 문제까지 모두 계산해야 함.

저장을 하면, 재귀의 **지수 시간**($O(2^n)$)이 **선형 시간**($O(n)$)으로 단축.     
단, 피보나치는 $n$을 구하기 위해 $0 \sim n-1$이 전부 필요하므로 사실상 두 방식의 계산량 **차이가 없음**.

## 4. `fib(100)` 의사결정

```
1단계(초급): 재귀 → 중복 계산 발생, O(2^n).

2단계(중급): 배열에 저장 (Bottom-Up DP) → 시간 O(n), 공간 O(n). 반복 호출 시 응답 시간 O(1).

3단계(상급): 변수 2개 → "fib(100)을 한 번만" 구할 거라면 배열도 불필요, 변수 2개로 공간 O(1).  반복 호출 시 응답 시간 O(n).
```

> **핵심**: 3단계는 면접관이 요청하기 전에 **스스로 trade-off를 인식하고 제안**하는 것.

## 5. 호출 패턴에 따른 최적 전략

같은 `fib(n)`이라도 어떻게 호출되느냐에 따라 최적 전략이 달라짐.

| 호출 패턴          | 최적 전략 | 시간 복잡도 (최초/이후)  | 공간 복잡도 |
|----------------|-------|-----------------|--------|
| `fib(n)` 단발    | 변수 2개 | $O(n)$ /  $O(n)$     | $O(1)$ |
| `fib(n)` 반복 | 배열 캐싱 | $O(n)$ / $O(1)$ | $O(n)$ |

여기가 헷갈렸다: '마지막 두 가지 경우만 고려한다'면 결국에는 마지막 단계에 이르는 방법은 2가지잖아ㅋㅋㅋ

'**마지막 단계에 도달하는 방법의 가짓수**'와 '**그 지점까지 도달하기 위해 쌓아온 전체 경로의 가짓수**'를 혼동.     
그런데 알고리듬이 구하고자 하는 것은 '마지막에 몇 칸을 움직였느냐'가 아니라, '**거기까지 오기 위해 거쳐온 서로 다른 모든 길의 개수**'.

### [단계2] '마지막 계단이 한 칸인 경우와 두 칸인 경우의 수를 나눈다'???

```text
직접 경우의 수를 따지면 '마지막 단계에 이르는 경우의 수의 합'이라는 개념이 이해가 되는데, 일반화하면 감이 안 옴.

예를 들어, 
5칸짜리 계단을 1칸 또는 2칸씩 올라가는 경우의 수:
- (1,1,1,1,1) 1가지
- (1,1,1,2) 4C3 해서 이건 4가지
- (1,2,2) 3C1 해서 이건 3가지
총 8가지
```

- **전체 경우: 8가지**     
(1,1,1,1,1)     
(1,1,1,2), (1,1,2,1), (1,2,1,1), (2,1,1,1)     
(1,2,2), (2,1,2), (2,2,1)

- **마지막이 1칸으로 5번째 계단에 오르는 경우의 수: 5가지**       
(1,1,1,1,1)     
(1,1,2,1), (1,2,1,1), (2,1,1,1)     
(2,2,1)

- **마지막이 2칸으로 5번째 계단에 오르는 경우의 수: 3가지**       
(1,1,1,2)     
(1,2,2), (2,1,2)

### [단계3] 뭔가 이해가 될 것 같아서 간질간질하다. 1/2/3칸 실험해보자.

```text
오, 뭔가 알 것 같다.
그러면 이제 n=4일 때, 계단을 1/2/3칸씩 오르는 것이 가능한 경우

(1,1,1,1) 1가지
(1,1,2) 3가지
(1,3) 2가지
(2,2) 1가지
총 7가지

* 마지막이 1인 경우: 1+2+1+0 = 4가지
* 마지막이 2인 경우: 0+1+0+1 = 2가지
* 마지막이 3인 경우: 0+0+1+0 = 1가지

그러니까 fib(4) = fib(3) + fib(2) + fib(1) = 4 + 2 + 1 = 7
```

### [단계4] 그런데 마지막 두 값이 하늘에서 뚝 떨어져?

```text
그런데 결국에는 fib(100)을 계산하려면 fib(0)부터 알아야 하는 거잖아? 누적 계산해야 하잖아?
그런데 무슨 '마지막 두 개의 값만 있으면' 된다는 거야? 그게 어디서 뚝 떨어져?
```

**CLAUDE**:     
맞아요. 그래서 DP(Dynamic Programming)가 존재하는 거예요.      
`fib(0)`부터 차근차근 올라가면서 값을 저장해두는 거예요.

밑에서부터 쌓아올려서 `fib(100)`까지 도달하는 거죠. 이게 Bottom-Up DP예요.     
반대로 `fib(100)`에서 시작해서 내려가는 방법도 있는데, 그게 재귀 + 메모이제이션, Top-Down DP고요.     
둘 다 결국 "한 번 계산한 값은 저장해두고 재활용한다"는 게 핵심이에요.     


### [단계5] 그럼 이미 다 계산해놓은 건...?

```text
그러면 계산해 놓은 거 그냥 다 갖다 버리고 마지막 2개만 기억하는 거네ㅋㅋㅋㅋ
그런데 그렇게 되면 fib(n) 실행할 때마다 계산해야 하는 거잖아?
```

**CLAUDE**:
ㅋㅋㅋㅋㅋ 맞아요! 그게 공간 최적화예요.     
어차피 직전 두 값만 필요하니까 나머지 다 버리는 거죠. 그래서 공간복잡도가 $O(n)$ → $O(1)$이 되는 거고요.     

저장 안 하면: `fib(100)` 호출할 때마다 처음부터 다시 계산해요. 매번 $O(n)$.     
memo 저장하면: 한 번 계산해두면 그다음부턴 $O(1)$로 바로 꺼내 써요.     

그래서 trade-off예요.     

> 공간 아끼려면 마지막 두 값만 유지, 대신 매번 계산.     
> 시간 아끼려면 전부 저장, 대신 공간 $O(n)$ 사용.     

"시/공간 중 시간 최적화를 우선하겠습니다" 이 말이 바로 이 선택의 순간에 하는 말이에요 ㅋㅋㅋ

n == 1 (1가지)     
    - 2(0), 1(1) -> 1가지


n == 2 (2가지)     
    - 2(0), 1(2) -> 1가지     
    - 2(1), 1(0) -> 1가지     


n == 3 (3가지)     
    - 2(0), 1(3) -> 1가지     
    - 2(1), 1(1) -> 2가지     


n == 4 (5가지)     
    - 2(0), 1(4) -> 1가지     
    - 2(1), 1(2) -> 3가지 (3C1)     
    - 2(2), 1(0) -> 1가지     


n == 5 (8가지)     
    - 2(0), 1(5) -> 1가지     
    - 2(1), 1(3) -> 4가지 (4C1)     
    - 2(2), 1(1) -> 3가지 (3C2)     


n == 6 (13가지)     
    - 2(0), 1(6) -> 1가지     
    - 2(1), 1(4) -> 5가지 (5C1)     
    - 2(2), 1(2) -> 6가지 (4C2)     
    - 2(3), 1(0) -> 1가지     


> 패턴 나왔다: 피보나치 수열이다.     
> 그런데 왜 피보나치 수열임?  

> $k(\text{2보 횟수}) + (n - 2k)(\text{1보 횟수}) = \mathbf{n - k}$     
> "계단은 $n$칸이지만, 네가 실제로 발을 내딛는 횟수는 $n-k$번이야. 그러니 $n-k$개 자리 중에서 $2$를 넣을 $k$곳을 골라봐!"     
> 그래서 $_{n-k}C_k$ 

나 진짜 졸업장 반납해야 할 듯... 이해 안 감...
그러니까 쉽게 풀어보자

> 1. 계단은 총 10칸($n$)
> 2. 2칸씩 3번($k$) 이동
> 3. 남은 1칸 횟수 = 10 - 2 * 3 = 4번
> 4. 총 이동 횟수 = 3(2칸 횟수) + 4(1칸 횟수) = 7
> 5. 총 이동 횟수 = $k + (n - 2k) = \mathbf{n - k}$
> 6. $_{n-k}C_k$ :  총 $(n-k)$회의 전체 횟수에서, $k$(2보 횟수)를 어디에 끼워넣을지 

> 총 가짓수를 $S_n$이라고 하자. (내가 찾은 수식: $S_n = \sum \binom{n-k}{k}$)
> 
> 
>  $S_n$을 구할 때, 마지막 걸음이 어떠했느냐에 따라 두 그룹으로 나뉨.
> 
> 1. 마지막이 1보 전진인 경우:     
> 마지막 1보를 제외한 나머지 $(n-1)$계단을 오르는 모든 조합의 수와 같음.      
> 이 값은 정의에 의해 **$S_{n-1}$**.
> 2. 마지막이 2보 전진인 경우:     
> 마지막 2보를 제외한 나머지 $(n-2)$계단을 오르는 모든 조합의 수와 같음.     
> 이 값은 정의에 의해 **$S_{n-2}$**.
>
> 결국 $S_n = S_{n-1} + S_{n-2}$ 가 성립.

### '마지막 단계' 거꾸로 생각하기

$1$층부터 올라가지 않고, **맨 마지막 옥상**($n$)에 서 있는 자신을 상상.     

"내가 여기 어떻게 왔지? 아, $n-1$에서 한 칸 왔거나, $n-2$에서 두 칸 왔겠네?"     

이 생각이 드는 순간 $f(n) = f(n-1) + f(n-2)$라는 식이 머릿속에 박히고, "어? 이거 피보나치네?"라고 결론 내리는 데 3초도 안 걸림.

> 그런데 난 여전히 안 와닿음


1. 왜 안 와닿을까? (본인의 사고 vs 점화식 사고)

본인의 사고 (조합론적): "2칸을 몇 번 섞느냐에 따라 전체 그림이 달라지잖아? $k=1$일 때랑 $k=2$일 때랑 판이 아예 다른데, 이걸 어떻게 뭉뚱그려?" -> (정밀한 설계자 마인드)     

점화식 사고 (재귀적): "중간에 $k$를 몇 번 썼든 상관없어. 어쨌든 옥상 바로 직전 발자국은 '1칸 밑' 아니면 '2칸 밑' 둘 중 하나잖아? 그 외의 외계인은 없어." -> (단순한 분류가 마인드)



2. 와닿게 만드는 '결계' 비유

옥상($n$층) 바로 앞에 두 개의 문이 있다고 상상해 보세요.

A문: $n-1$층에서 들어오는 문 (무조건 1칸 걸어 올라와야 함)

B문: $n-2$층에서 들어오는 문 (무조건 2칸 점프해서 들어와야 함)

옥상에 도착한 모든 사람은 반드시 이 두 문 중 하나를 통과해서 들어왔습니다.

그렇다면 "옥상에 도착하는 총 방법의 수"는 당연히 **"A문으로 들어온 사람 수 + B문으로 들어온 사람 수"**가 되겠죠?


A문으로 들어온 사람들은 전부 $n-1$층까지 오느라 고생한 사람들이니, 그 가짓수는 $S_{n-1}$입니다.

B문으로 들어온 사람들은 전부 $n-2$층까지 오느라 고생한 사람들이니, 그 가짓수는 $S_{n-2}$입니다.

중간에 이 사람들이 2칸을 몇 번 섞었는지는 중요하지 않습니다. **"마지막 관문을 통과하기 직전의 위치"**가 어디였냐만 따지는 거예요.



3. 본인이 찾은 $\sum \binom{n-k}{k}$와 연결하기

본인이 찾은 수식이 대단한 이유는, 저 $S_{n-1}$과 $S_{n-2}$라는 뭉뚱그려진 덩어리 내부를 현미경으로 다 까본 것과 같기 때문입니다.

$S_{n-1}$ 안에는 $k=0, 1, 2...$인 시나리오가 다 들어있고,$S_{n-2}$ 안에도 $k=0, 1, 2...$인 시나리오가 다 들어있습니다.

이걸 다 합치면 결국 본인이 구한 $n$층의 시나리오 합계와 수학적으로 완벽히 일치하게 됩니다.

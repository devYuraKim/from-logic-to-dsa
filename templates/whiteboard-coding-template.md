# Whiteboard Coding Template

## I. Common Principles

- **Think Aloud**: 현재 생각, 선택 이유, 다음 행동을 지속적으로 구두 설명.
- **Proactive Communication**: 핵심 의사 결정 지점에서 면접관과 방향을 확인.

  **&lt;Communication guide&gt;**

  | 단계                          | 인터랙션 수준         | 핵심 체크리스트                                                                                                                                                             |
  | :---------------------------- | :-------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | **1. Clarification**          | **🟩 최상 (주도적)**  | 입력 데이터의 범위, 중복 여부, 빈 값(`null`) 처리 등 힌트 수집.                                                                                                             |
  | **2. Strategy & Structuring** | **🟩 상 (방향 확인)** | "이 자료 구조와 접근 방식으로 진행하려고 합니다. 괜찮을까요?", "시/공간 중 시간 최적화를 우선하겠습니다." 등의 질문으로 면접관과 **함께** DSA 결정.                         |
  | **3. Pseudo Code**            | **🟩 상 (논리 확정)** | 구현 전 흐름 정리 후 “이 방향으로 코드 구현해도 될까요?” 최종 승인 확인.                                                                                                    |
  | **4. Implementation**         | **하 (수행)**         | 지나친 질문은 구현 능력 의심 위험. THINK ALOUD(현재 상태와 의도 전달) 유지하되, 막히거나 불확실한 경우에만 확인.                                                            |
  | **5. Review & Optimization**  | **🟩 상 (심층적)**    | "현재는 $O(n^2)$인데, HashMap을 활용하면 $O(n)$으로 줄일 수 있습니다. 다만 공간 복잡도가 $O(n)$ 추가됩니다", "테스트 케이스를 더 추가해 볼까요?" 등 분석 및 개선 의지 표현. |

- **Recovery Protocol**:
  1. **Report**: 침묵 방지 + 상황 통제  
     막힌 지점을 면접관에게 객관적으로 공유하고, 현재 상태를 명확히 표현.  
     &gt; "이 조건문에서 무한 루프 가능성이 보여 잠시 검토하겠습니다"
  2. **Recap & Reset**: 논리적 사고 견지 + 구조 복구  
     지금까지의 접근법과 가정을 요약하고, 필요 시 Step1(Clarification)으로 돌아가 문제 정의와 제약 사항을 재확인.  
     &gt; "현재는 HashMap으로 O(1) lookup을 가정했는데, 혹시 놓친 조건이 있는지 다시 확인해보겠습니다."
  3. **Reduce**: 실제 문제 해결 착수  
     문제를 가장 작은 입력값 혹은 단순한 예제로 축소하여 흐름 재구성.  
     &gt; "간단한 케이스로 다시 시뮬레이션 해보겠습니다."
  4. **Realign**: 협업 시그널 + 방향 확정  
     수정된 가설을 면접관에게 공유하고 방향 재설정.  
     &gt; "이 방향으로 계속 진행해도 괜찮을까요?"

## II. 5 Step Framework

#### 1. Clarification (문제 명확화 및 제약 사항 확인)

문제를 확인 후, 질문을 통해 범위 좁히기.

- **Input/Output 확인**: "입력값의 자료형은 무엇인가?", "빈 리스트가 들어올 수 있는가?"
- **Constraints 확인**: "시/공간 복잡도의 제한이 있는가?", "데이터 크기(n)는 어느 정도인가?"
- **Edge cases 검토**: "값이 하나일 때, 중복될 때, 혹은 null일 때 어떻게 처리할 것인가?"

#### 2. Strategy & Structuring (전략 수립 및 구조화)

- **Data Structure, Algorithm 선택 및 근거 제시**: "연속된 메모리가 필요 없으므로 Linked List 선택", "HashMap으로 lookup을 O(1)로 최적화"
- **Visualization**: 간단한 도식화로 흐름 시뮬레이션

#### 3. Pseudo Code (의사코드 작성)

- 문법 없이 논리적 흐름만 작성
- 논리 단계별로 설명할 것
- 단계별 예상값 주석 처리하여 명시

#### 4. Implementation (실제 코딩)

- 불확실한 문법이나 함수는 가정 후 진행
- 의미 있는 네이밍 사용
- 필요 시 메서드 분리 제안

#### 5. Review & Optimization (검토 및 최적화)

- Dry run으로 예제 시뮬레이션
- 시간/공간 복잡도 분석
- Edge case 재검증
- 문제 발견 시 즉시 원인 설명 후 수정, 혹은 면접관의 의견 질문

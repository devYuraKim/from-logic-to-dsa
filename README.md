# from-logic-to-dsa

**Algorithms are not just patterns to memorize, but logic to be engineered.**  

This repository documents my journey of learning **Data Structures and Algorithms** by starting from **raw logic**, not pre-learned patterns.
Instead of immediately applying known solutions, I first attempt to solve each problem using my own reasoning, then identify the limitations and learn the appropriate data structure or algorithm.

## Methodology 

1. Approach the problem with my own logic (brute force is fine)
2. Identify limitations in my approach (time/space complexity, edge cases)
3. Learn the appropriate data structure or algorithm
4. Refactor and optimize the solution

> The goal is not just to solve problems, but to understand **why** certain approaches work better than others.

## Goal

To develop strong problem-solving intuition by bridging the gap between **raw thinking** and **structured algorithms**.

**알고리듬은 단순히 암기해야 할 패턴이 아니라 설계해야 할 논리입니다.**

이 저장소는 기존의 패턴을 답습하는 대신, **순수 논리**에서 시작하여 **자료구조와 알고리즘**을 체득해 나가는 과정을 기록합니다.
이미 알려진 해답을 바로 적용하기보다, 먼저 나만의 논리로 문제를 해결해 본 뒤 그 한계를 분석하고 적절한 자료구조나 알고리즘을 학습하여 보완합니다.

## 방법론

1. 나만의 논리로 문제에 접근 (브루트 포스여도 무관)
2. 내 접근법의 한계 파악 (시/공간 복잡도, 예외 상황)
3. 적절한 자료구조나 알고리듬 학습
4. 솔루션 개선 및 최적화

> 단순히 문제를 푸는 것이 아니라, **왜** 특정 접근 방식이 다른 방식보다 더 효율적인지 이해하는 것을 목표로 합니다.

## 목표

**가공되지 않은 생각**과 **정교한 알고리듬** 사이의 간극을 메워 문제 해결력을 위한 직관을 기릅니다.
